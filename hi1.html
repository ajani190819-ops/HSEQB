Help & Tutorial

Getting Started

Open the sidebar and work through each section in order: Session → Players → Teams → Categories. Once configured, use the Record Answer panel to log answers in real time. All data syncs automatically across connected devices via Firebase.

Sessions

Each match is a Session. Create one with a descriptive name before recording answers. Use Load Session to switch between matches. Sessions can be excluded from analytics — excluded sessions are dimmed and ignored in all Stats calculations.

Players & Teams

Add players in the Players section, then create teams in Teams and assign players to them. Only players assigned to a team are selectable when recording answers. Players are stored globally and available across all sessions.

Recording Answers

Select a Player, then a Point Type, then a Category (optional for Neg), then press Record Answer. Use the answer log to delete mistakes.

Toss-Ups Heard Counter

Tracks how many toss-up questions have been read. Increments automatically on every Power, Toss-up, or Neg (a Neg immediately followed by a correct answer counts as one question). Use the +/− buttons to correct it manually. Required for all per-player efficiency metrics.

Stats (Analytics)

Toggle to the Stats view using the switch in the header. Stats are aggregated across all non-excluded sessions and organized into four tabs:

  • Overview — totals, category breakdowns, top performers
  • Players — ranked leaderboard; expand for metrics or click name for detail
  • Charts — visual breakdown of answer types, categories, and trends
  • Team Compositions — per-team stats per session

Player Metrics

Expand any player row to see individual metrics. Per-TU-heard stats require toss-ups heard data to be set.

  • Points — total points across all sessions
  • Toss-ups — total correct answers (powered counts as Power, marked with a bolt); Power = +15 before the mark, regular = +10 after
  • Actual Pts / 20 TU — earned points per 20 toss-ups heard: pts / TH × 20
  • TU Efficiency — pts per correct answer: (15P + 10T) / (P+T). Range 10–15. Blind to negs, misses, and TU heard.
  • Accuracy — correct buzz rate: (P+T) / (P+T+N+M) × 100. Blind to TU heard.
  • Conversion Rate — correct answers per TU heard: (P+T) / TH × 100
  • Skill Score — TU Efficiency + Accuracy, normalized so team avg = 100
  • Contribution Score — Conversion Rate, normalized so team avg = 100
  • Overall Score — 50/50 average of Skill and Contribution

Score System

All scores are relative to the team — the team average always equals 100. A player at 150 is 50% above average; a player at 50 is half the team average. Each score is regression-adjusted using TU Heard — players with more data have their numbers trusted more.

Skill = (TU Efficiency + Accuracy) / 2  ·  Contribution = Conversion Rate  ·  Overall = (Skill + Contribution) / 2

Players below the low-data threshold are excluded from scoring by default. Toggle inclusion in the Low-Data Players panel in the sidebar.

Leaderboard Sorting

Sort by Points, Overall Score, Pts / 20 TU, Actual Pts / 20 TU, TU Efficiency, Accuracy, Powers, Negs (fewest first), or A–Z. Ties in Points are broken by most Powers, then fewest Negs.

Export & Import

Export to CSV for spreadsheets or JSON to back up and re-import a full session. Both options are in the Data section of the sidebar.

What Syncs Globally

Stored in Firebase and synced instantly across all connected devices:

  • All session data (answers, scores, toss-ups heard)
  • Global player pool
  • Skill score settings and low-data overrides
  • App version and update info

Stored locally only (not synced): Dark mode, accent color, scrollbar visibility, sidebar state.

Version Badge & Deploy Indicator

The version badge in the header shows the current file version. When viewed on the live GitHub Pages site, its border changes color to reflect the latest deploy status — green = deployed successfully, yellow pulsing = deploy in progress, red = deploy failed. If a new deploy completes while you have the page open, a banner will appear prompting you to refresh.

Updates (Local File)

If you are running a local copy of the file, an update banner will appear when a newer version has been published. Click Download Update to get the latest file. Users on the GitHub Pages site receive updates automatically on refresh.
